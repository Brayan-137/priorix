# Priorix

### Requerimientos funcionales
1. Gestión de eventos y tareas

Permite agregar eventos como tareas, proyectos, etc. a un calendario donde se pueda visualizar la fecha límite de entrega de dicho trabajo.

Junto a esto debe poder definir el tiempo estimado de trabajo que conllevaría realizar la asignación, la prioridad.

Se pueden clasificar por medio de etiquetas.

Se puede reagendar tareas o organizarlas de forma manual.

### 2. Planificación automática del tiempo

Se definirá horarios en los que realizar los trabajos y automaticamente se repartiran en esos horarios los diferentes espacios que se designen para desarrollarlo a partir de la cantidad máxima de tiempo por sesión o de sesiones definidas por el usuario.

Habrán actividades fijas las cuales no moveran al momento de planificar el desarrollo de las actividades. Para esto se podrán definir si estas activides se repiten de forma semana y si es así, hasta que fechas.

Alertará de forma inmediata al usuario y sus configuraciones para organizar los espacios de trabajo de la actividad es incombatible y si necesita cambiar la configuración para poder cumplir con su desarrollo.

### 3. Seguimiento y estadísticas

Habrá un registro y estadística de la culminación de las tareas para incentivar a los usuarios a ser más disciplinados. Por lo cual habrá formas en las que se podrá visualizar esta constancia, como:

El calendario mostrando en rojo las tareas hechas y las no hechas (En discusión)

Graficos con el porcentaje de realización de las diferentes actividades, por día, semana y mes.

Racha de los días donde se completaron todas las tareas

La evolución de la mascota, tipo pokemon. Basada en un sistema de experiencia por utilizar el modo de enfoque pero principalmente por completar tareas.

### 4. Modo de concentración y descansos

Habrá un modo de concentración donde se configurará un crónometro para marcar el fin de la tarea.
